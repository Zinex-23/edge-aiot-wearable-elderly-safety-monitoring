# Nhật ký làm việc — 2026-05-20

## Tổng quan

Ngày tập trung hoàn toàn vào **Cloud Integration** và **Bug Fix** trên toàn bộ stack: Edge firmware, Android app, và Flask/MongoDB backend. Đây là ngày đánh dấu hệ thống lần đầu tiên hoạt động **end-to-end thật sự**: dữ liệu từ ESP32 → BLE → Android → Render (HTTPS) → MongoDB Atlas.

---

## 1. Cloud Backend (Flask / MongoDB)

### 1.1 Thiết kế lại server.py
- Chuyển `MONGO_URI` sang `os.environ.get("MONGO_URI")` — sẵn sàng cho Render deploy
- `PORT` đọc từ env (Render inject `PORT=10000`)
- Thêm 3 collection mới: `users`, `vitals`, `fall_events`

### 1.2 Thêm API mới
| Endpoint | Method | Chức năng |
|---|---|---|
| `/api/auth/register` | POST | Tạo tài khoản, lưu `passwordHash` SHA-256 |
| `/api/auth/login` | POST | Xác thực, trả về profile |
| `/api/vitals` | POST | Nhận dữ liệu sức khỏe từ app |
| `/api/vitals` | GET | Truy vấn theo `userId`, `range=1h/24h` |
| `/api/fall_event` | POST | Lưu sự kiện té ngã |
| `/api/fall_events` | GET | Lấy lịch sử té ngã |

### 1.3 Deploy lên Render
- URL: `https://edge-aiot-wearable-elderly-safety.onrender.com`
- Start Command: `python server.py` (Flask dev server trên port từ env)
- Tạo `requirements.txt`: `flask`, `pymongo[srv]`, `dnspython`, `gunicorn`

## 2. Android App — Cloud Integration

### 2.1 Dependencies & Permissions
- Thêm `OkHttp 4.12.0` + `Gson 2.10.1` vào `build.gradle.kts`
- Thêm `INTERNET`, `ACCESS_NETWORK_STATE` vào `AndroidManifest.xml`
- Tạo `network_security_config.xml` cho phép HTTP cleartext khi test localhost

### 2.2 CloudApi.kt (mới)
- HTTP client dùng OkHttp, `RENDER_URL` là 1 constant dễ đổi
- Hàm: `login()`, `register()`, `postVital()`, `getVitals()`, `postFallEvent()`, `isServerReachable()`

### 2.3 Auth screens
- `LoginScreen`: gọi `CloudApi.login()` thay vì hardcode; demo `000/000` vẫn local
- `RegisterScreen`: gọi `CloudApi.register()`; thêm ô xác nhận mật khẩu; show/hide password cho cả 2 màn

### 2.4 MonitoringViewModel
- Upload vitals lên cloud mỗi lần nhận BATCH packet (lấy average của batch)
- Dùng `System.currentTimeMillis()` làm timestamp — tránh lỗi timestamp epoch 1970 từ ESP32 boot time
- Fetch 1h history mỗi 5 phút; fetch 24h history mỗi 30 phút
- Offline fallback: cache vào SharedPreferences, load lại khi mất mạng
- `deviceId` = BLE MAC address (`BleState.Connected.deviceAddress`)

### 2.5 Data bucketing
- Cloud trả về raw list (có thể 100+ điểm)
- `bucketCloud()`: chia thành `12 slot × 5min` (1h) hoặc `24 slot × 1h` (24h), average per slot
- Chart hiển thị đúng mật độ: 12 điểm cho 1h, 24 điểm cho 24h

### 2.6 MonitoringScreen
- Loading animation: card xoay `Sync` icon + shimmer bar bằng `Canvas` (tránh `LinearProgressIndicator` crash trên Android 10 với BOM 2024.01.00)
- Caregiver không thấy banner "No device connected"

## 3. Bug Fixes

| Bug | Nguyên nhân | Fix |
|---|---|---|
| App crash khi nhấn Login | `CircularProgressIndicator` dùng `keyframes.at()` không tương thích Material3 trên Android 10 | Thay bằng `Icon(Refresh) + rotate()` |
| App crash khi chọn tab 1h | `LinearProgressIndicator` cùng nguyên nhân | Thay bằng `Canvas` shimmer custom |
| Compile error `No set method` | `LongArray[idx] += value` trong lambda `.let{}` không được Kotlin compiler hỗ trợ | Đổi sang `if` block + explicit assignment |
| Chart 1h quá nhiều điểm (~100+) | `fetchCloudVitals` flatten raw list, không bucket | Thêm `bucketCloud()` → 12/24 điểm cố định |
| 24h chart trống | 405 documents có timestamp 1970 (từ ESP32 boot time cũ) không lọc được | Xóa 415 doc bẩn; fix timestamp dùng `System.currentTimeMillis()` |
| `startLiveTick()` ghi đè cloud data | Live tick mỗi giây dùng `vitalsStore` thay vì cloud data | Ưu tiên `cloud1hHr/cloud24hHr` trong tick |
| Login báo sai mật khẩu | `CircularProgressIndicator` crash trước khi request gửi đi | Fix crash → login hoạt động bình thường |
| Query 24h lấy theo deviceId | Filter dư `deviceId` loại bỏ data từ device khác | Bỏ filter deviceId, chỉ dùng `userId` |
