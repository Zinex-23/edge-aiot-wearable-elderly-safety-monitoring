## 1. Firmware ESP32-S3 (`S3_BLE_test_2`)

### 1.1 Fix bug NimBLE setValue (CRITICAL)
- **Vấn đề**: `setValue(const char*)` trigger template overload `setValue<T>(const T&)` với T=`const char*` → gửi 4 byte địa chỉ pointer thay vì string
- **Triệu chứng**: Android nhận được `ꀀ??` (garbage bytes) thay vì CSV payload
- **Fix**: Thêm helper `setStr()` dùng `setValue((uint8_t*)s, strlen(s))`
- **Tác động**: Toàn bộ BLE data transmission bắt đầu hoạt động đúng

### 1.2 Fix LEDC initialization lỗi
- **Vấn đề**: `E (818) ledc: ledcSetup failed!` — `tone(PIN_BUZZER, 1)` với 1Hz quá thấp cho ESP32-S3
- **Fix**: Đổi thành `tone(PIN_BUZZER, BUZZER_FREQ_HZ)` (2300 Hz)

### 1.3 Fix upload timeout
- **Vấn đề**: PlatformIO "Write timeout" khi flash
- **Fix**: Thêm `upload_protocol = esp-builtin` vào `platformio.ini`

### 1.4 Fall Detection Pipeline — 7 tầng lọc hoàn chỉnh

Thiết kế và tinh chỉnh toàn bộ pipeline:

```
IMU 50Hz → [1] CANDIDATE → [2] ACTIVITY → [3] HIGH-IMPACT
        → [4] IMPACT CHECK → [5] AI WINDOW (6s) → [6] FALL FSM → [7] ALARM
```

**Thông số ngưỡng cuối cùng:**

| Tham số | Giá trị |
|---|---|
| `CANDIDATE_GYRO_THRESHOLD` | 240 dps |
| `ACTIVITY_ACC_THRESHOLD` | 2.0 g |
| `ACTIVITY_GYRO_THRESHOLD` | 50 dps |
| `HIGH_IMPACT_ACC_MIN` | 2.0 g |
| `HIGH_IMPACT_GYRO_MIN` | 300 dps |
| `FALL_DECISION_THRESHOLD` | 0.42 |
| `AI_WINDOW_DURATION_MS` | 6000 ms (3 windows) |
| `CANCEL_ACC_THRESHOLD` | 3.5 g |
| `CANCEL_GYRO_THRESHOLD` | 150 dps |
| `FALL_STILL_DURATION_MS` | 5000 ms |
| `FALL_MONITOR_TIMEOUT_MS` | 10000 ms |

**Model TFLite V84:** F1=93.45%, Recall=98.51%, Threshold=0.42

### 1.5 AI Window mechanism
- Peak mở "công tắc" AI trong 6s
- Trong 6s: chạy inference mỗi 2s (3 windows)
- Hết 6s không có fall → tắt, reset `highImpactSeen`
- Fall trong 6s → đóng window, FSM tiếp quản

### 1.6 Fall State Machine (FSM)
```
FDS_IDLE → [AI fall] → FDS_FALL_WATCH (5 windows = 10s)
  → [cử động mạnh] → HUỶ về IDLE
  → [hết 10s] → FDS_STILL_TIMING (10s monitor)
    → [nằm im 5s liên tục] → ALARM
    → [hết 10s monitor] → SAFE, về IDLE
```

### 1.7 Sliding stillness timer
- Cử động trong STILL_TIMING reset timer (không cancel state)
- Nằm im đủ 5s liên tục → ALARM
- Hết 10s monitor window → coi là SAFE

### 1.8 LED State Machine (7 states)
```
LED_IDLE → LED_MOVING → LED_ACTIVE → LED_CANDIDATE
→ LED_AI_FALL → LED_BLINK_WATCH → LED_ALARM
```
LED phản ứng trực tiếp với pipeline stage, không dùng nút bấm để đổi state.

### 1.9 Button dual mode
- Bình thường: nhấn nút → SOS thủ công (gửi ALERT ngay)
- Đang alarm: nhấn nút → cancel alarm, gửi SAFE

### 1.10 BMI snapshot broadcast
- Thêm packet `BMI,<seq>,<ts>,<peakAcc>,<peakGyro>,<active>` mỗi 5s
- App nhận để hiển thị sensor activity real-time

## 2. BLE Protocol

### 2.1 Protocol 4 packet types (CSV ASCII)

| Packet | Characteristic | Cadence | Format |
|---|---|---|---|
| `ALERT` | ALERT (notify) | On fall | `ALERT,seq,ts,fall,status,fall_prob,non_fall_prob` |
| `SAFE` | ALERT (notify) | On button | `SAFE,seq,ts` |
| `BATCH` | VITALS (notify) | 25s | `BATCH,seq,hr0\|...\|hr4,spo2_0\|...,ts0\|...` |
| `BMI` | VITALS (notify) | 5s | `BMI,seq,ts,peak_acc_g,peak_gyro_dps,active` |

### 2.2 READY handshake
- Android ghi `"READY"` vào CONTROL characteristic sau khi subscribe
- ESP32 reply `"ACK:READY"` → bắt đầu gửi data
